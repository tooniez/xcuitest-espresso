# iOS Setup

Native iOS app and XCUITest suite are generated by `scripts/scaffold-ios.sh` using [XcodeGen](https://github.com/yonaskolb/XcodeGen).

## What gets created

- **SwiftUI app** (`DemoApp`) with accessibility identifiers matching test assertions
- **UI test target** (`DemoAppUITests`) with a smoke test for home screen content
- **`project.yml`** — source of truth for the Xcode project
- **`DemoApp.xcodeproj`** — generated when XcodeGen runs during scaffold or test run

## Requirements

- macOS with Xcode (Command Line Tools + iOS Simulator)
- XcodeGen: `brew install xcodegen`

## Scaffold

```bash
./scripts/scaffold-ios.sh
```

If XcodeGen was not installed during scaffold:

```bash
brew install xcodegen
cd ios && xcodegen generate
```

Use `--force` to regenerate Swift sources and `project.yml` from the script templates.

## Run tests locally

```bash
./scripts/run-ios.sh
```

Customize simulator in `.env` (copy from `.env.example`):

```bash
cp .env.example .env
# edit IOS_SIMULATOR_NAME and IOS_SIMULATOR_OS
./scripts/run-ios.sh
```

List available simulators:

```bash
xcrun simctl list devices available
```

Equivalent `xcodebuild` invocation:

```bash
xcodebuild test \
  -project ios/DemoApp.xcodeproj \
  -scheme DemoApp \
  -destination 'platform=iOS Simulator,name=iPhone 16,OS=18.0' \
  -resultBundlePath reports/ios/DemoApp.xcresult \
  CODE_SIGNING_ALLOWED=NO
```

## Opening in Xcode

```bash
open ios/DemoApp.xcodeproj
```

Select the **DemoApp** scheme and press `Cmd+U` to run UI tests from the IDE.

## Debug commands (`simctl` / `xcodebuild`)

Useful while running XCUITests or debugging **Get Weather** traffic with [Charles Proxy](charles-proxy.md):

```bash
xcrun simctl list devices available
xcrun simctl bootstatus booted
xcrun simctl launch booted com.example.mobileuiautomationdemo
xcrun simctl terminate booted com.example.mobileuiautomationdemo
```

Run only the weather XCUITest (one Open-Meteo request for Charles):

```bash
xcodebuild test \
  -project ios/DemoApp.xcodeproj \
  -scheme DemoApp \
  -destination 'platform=iOS Simulator,name=iPhone 16,OS=18.0' \
  -only-testing:DemoAppUITests/DemoAppUITests/testGetWeatherShowsSanFranciscoTemperatureInFahrenheit \
  -resultBundlePath reports/ios/DemoApp.xcresult \
  CODE_SIGNING_ALLOWED=NO
```

After `./scripts/run-ios.sh`, open failures in Xcode:

```bash
open reports/ios/DemoApp.xcresult
```

Stream app logs while tapping **Get Weather** in the Simulator:

```bash
xcrun simctl spawn booted log stream --level debug \
  --predicate 'process == "DemoApp"'
```

Full Charles + proxy setup: [charles-proxy.md](charles-proxy.md#6-debug-commands-charles--tests).

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `DemoApp.xcodeproj` missing | Run `cd ios && xcodegen generate` |
| Simulator not found | Adjust `IOS_SIMULATOR_NAME` / `IOS_SIMULATOR_OS`, or rely on auto-detect in `run-ios.sh` |
| Signing errors in CI | `run-ios.sh` passes `CODE_SIGNING_ALLOWED=NO` for simulator-only runs |
| XcodeGen not installed | `brew install xcodegen` |

Full runbook: [Troubleshooting & learnings](troubleshooting.md).
