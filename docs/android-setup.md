# Android Setup

Native Android app and Espresso UI tests are generated by `scripts/scaffold-android.sh`.

## What gets created

- **Gradle Kotlin DSL** multi-module layout with `:app`
- **MainActivity** showing welcome and product copy aligned with `test-data/products.json`
- **Espresso test** `MainActivityEspressoTest` asserting visible text on the home screen
- **Gradle wrapper** (when Gradle is available on the machine running scaffold)

## Requirements

- JDK 17+
- Android SDK with `compileSdk` 34 and a system image for your emulator
- `ANDROID_HOME` or Android Studio SDK path configured

## Scaffold

```bash
./scripts/scaffold-android.sh
# Overwrite existing generated files:
./scripts/scaffold-android.sh --force
```

If `gradlew` is missing after scaffold, either:

```bash
cd android && gradle wrapper --gradle-version 8.7
```

or open `android/` in Android Studio and sync Gradle.

## Run tests locally

1. Start an AVD or connect a device with USB debugging enabled.
2. Verify `adb devices` lists the target.
3. Run:

```bash
./scripts/run-android.sh
```

Equivalent Gradle command:

```bash
cd android && ./gradlew connectedDebugAndroidTest
```

## Debug commands (`adb` / Gradle)

Useful while running Espresso tests or debugging **Get Weather** traffic with [Charles Proxy](charles-proxy.md):

```bash
adb devices -l
adb shell settings get global http_proxy          # expect 10.0.2.2:8888 when proxied
adb shell am start -n com.example.mobileuiautomationdemo/.MainActivity
adb shell am force-stop com.example.mobileuiautomationdemo
cd android && ./gradlew installDebug              # install without running tests
```

Run only the weather Espresso test (one HTTPS call — easy to spot in Charles):

```bash
cd android && ./gradlew connectedDebugAndroidTest \
  -Pandroid.testInstrumentationRunnerArguments.class=com.example.mobileuiautomationdemo.ui.MainActivityWeatherEspressoTest
```

Tail errors while Charles SSL proxying is enabled:

```bash
adb logcat -c && adb logcat *:E | grep -iE 'SSL|Certificate|Weather'
```

Full Charles + proxy setup: [charles-proxy.md](charles-proxy.md#6-debug-commands-charles--tests).

## Instrumentation

The app module sets:

```kotlin
testInstrumentationRunner = "androidx.test.runner.AndroidJUnitRunner"
```

Espresso dependencies live in `android/app/build.gradle.kts` under `androidTestImplementation`.

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `SDK location not found` | Set `ANDROID_HOME` or add `android/local.properties` with `sdk.dir=...` |
| No connected devices | Start emulator: `emulator -avd <name>` |
| Gradle wrapper missing | Run scaffold again with Gradle installed, or use Android Studio sync |
| Espresso `InputManager.getInstance` on API 36 | Upgrade to Espresso **3.7.0** and AndroidX Test **1.7.0** (see [troubleshooting.md](troubleshooting.md)) |
| Emulator disk space fatal | Free space on system volume before starting AVD |

Full runbook: [Troubleshooting & learnings](troubleshooting.md).

## Charles Proxy (weather API demo)

To capture **Get Weather** HTTPS traffic from an emulator, follow [charles-proxy.md](charles-proxy.md). Summary:

1. Start an AVD and set proxy to `10.0.2.2:8888` (Charles on the Mac host).
2. Install the Charles root CA on the emulator (debug builds trust user CAs).
3. Enable SSL Proxying for `api.open-meteo.com:443` in Charles.
4. Run `./scripts/run-android.sh` or tap **Get Weather** in the app.
