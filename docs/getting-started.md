# Getting Started

This repo demonstrates native mobile UI automation on Android (Espresso) and iOS (XCUITest). Use the scaffold scripts to generate both app projects, then run the platform test scripts locally or in CI.

## Prerequisites

| Platform | Tools |
|----------|-------|
| Android | JDK 17, Android SDK (API 34), emulator or device |
| iOS | macOS, Xcode, [XcodeGen](https://github.com/yonaskolb/XcodeGen) (`brew install xcodegen`) |
| Shared | Bash, Git |

## 1. Configure (optional)

```bash
cp .env.example .env
```

Edit `.env` to pin an iOS simulator, change the Android Gradle task, or override the app package. Empty simulator fields auto-detect the first available iPhone on macOS.

## 2. Scaffold native projects

Generate both platforms:

```bash
chmod +x scripts/*.sh scripts/lib/common.sh
./scripts/scaffold-all.sh
```

Or scaffold individually:

```bash
./scripts/scaffold-android.sh
./scripts/scaffold-ios.sh
```

Re-run with `--force` to overwrite generated files (for example after changing templates in the scripts).

## 3. Run UI tests

**Android** (requires a running emulator or connected device):

```bash
./scripts/run-android.sh
```

**iOS** (requires Xcode simulator):

```bash
./scripts/run-ios.sh
```

Set simulator in `.env`:

```bash
IOS_SIMULATOR_NAME="iPhone 15"
IOS_SIMULATOR_OS=17.5
```

## 4. Project layout after scaffold

```
android/
  app/                          # Kotlin app + Espresso tests
  gradlew
ios/
  DemoApp/                      # SwiftUI app
  DemoAppUITests/               # XCUITest target
  DemoApp.xcodeproj             # Generated by XcodeGen
  project.yml
test-data/                      # Shared JSON fixtures
reports/                        # Local test output (created on run)
scripts/
  scaffold-android.sh
  scaffold-ios.sh
  scaffold-all.sh
  run-android.sh
  run-ios.sh
```

## 5. CI

GitHub Actions workflows copy `.env.example` to `.env`, then call the same `run-*.sh` scripts. Ensure projects are scaffolded in CI or commit the generated native trees.
